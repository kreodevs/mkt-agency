# Modelo Juntos — Implementation Plan

> **Goal:** Implementar el flujo completo donde Hermes propone acciones (leads, posts, optimizaciones) → usuario aprueba/rechaza en frontend o chat → backend ejecuta.

**Architecture:** Sistema de `Proposal` como entidad central con payload tipado por acción, endpoints CRUD para frontend, webhook `/webhooks/hermes-proposal` para que Hermes inyecte propuestas, y servicio de ejecución que despacha a los módulos existentes.

**Tech Stack:** NestJS + TypeORM + PostgreSQL (backend), React + Kreo UI (frontend)

**Modules (orden de implementación):**
1. Backend: Proposal entity + module + CRUD
2. Backend: Webhook endpoint for Hermes proposals
3. Backend: Execution service (dispatch to Leads, Posts, Campaigns)
4. Backend: Webhook callback to notify Hermes on approval/rejection
5. Frontend: ProposalsPage + API service
6. Frontend: Integration in sidebar + AppLayout routing
7. Verification: full flow test

### Proposal Entity

```typescript
export type ProposalStatus = 'pending' | 'approved' | 'rejected' | 'executed' | 'failed';
export type ProposalActionType = 'create_post' | 'contact_lead' | 'score_lead' | 'optimize_campaign' | 'add_keyword' | 'pause_keyword' | 'create_campaign' | 'custom_message';

@Entity('proposals')
export class Proposal {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column()
  tenantId: string;

  @Column({ nullable: true })
  productId: string;

  @Column({ length: 50 })
  actionType: ProposalActionType;

  @Column({ type: 'jsonb' })
  payload: Record<string, any>;

  @Column({ type: 'text', nullable: true })
  rationale: string;  // Por qué Hermes propone esto

  @Column({ length: 20, default: 'pending' })
  status: ProposalStatus;

  @Column({ type: 'text', nullable: true })
  rejectionReason: string;

  @Column({ type: 'text', nullable: true })
  resultSummary: string;  // Resultado después de ejecutar

  @Column({ nullable: true })
  executedById: string;  // User ID que aprobó

  @Column({ nullable: true })
  executedAt: Date;

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;
}
```

### Service Methods

- `create(dto)` — creates pending proposal
- `findByTenant(tenantId, filters?)` — list with status/productId filter
- `approve(id, userId)` — validates, executes action, updates status
- `reject(id, reason?)` — marks as rejected
- `executeProposal(proposal)` — dispatches to correct handler based on actionType

### Controller Endpoints

- `GET /tenants/:tenantId/proposals` — list proposals (with ?status=pending&productId=xxx)
- `GET /tenants/:tenantId/proposals/:id` — get single proposal
- `POST /tenants/:tenantId/proposals` — create proposal (for internal use, but also webhook)
- `POST /tenants/:tenantId/proposals/:id/approve` — approve (userId from JWT)
- `POST /tenants/:tenantId/proposals/:id/reject` — reject with optional reason

### New Endpoint

`POST /webhooks/hermes-proposal`

Body:
```json
{
  "tenantId": "oraltrack",
  "productId": "uuid",
  "actionType": "create_post",
  "payload": { "content": "..." },
  "rationale": "Detectamos tendencia en X sobre..."
}
```

Response:
```json
{
  "received": true,
  "proposalId": "uuid",
  "status": "pending"
}
```

## Módulo 3: Execution Service

**Objective:** Cuando se aprueba una propuesta, ejecutar la acción real usando los servicios existentes.

**Files:**
- Create: `backend/src/proposals/proposal-executor.service.ts`
- Modify: `backend/src/proposals/proposals.module.ts` (add imports for LeadsModule, PostsModule, CampaignsModule)

### Action Handlers

- `create_post` → calls PostsService.create()
- `contact_lead` → calls LeadsService.updateStage() to 'contactado'
- `score_lead` → calls LeadsService.updateScore()
- `add_keyword` → calls CampaignsService.addKeyword()
- `pause_keyword` → calls CampaignsService.pauseKeyword()
- `create_campaign` → calls CampaignsService.create()
- `custom_message` → logs, marks executed (for manual actions)

## Módulo 4: Webhook Callback to Hermes

**Objective:** Cuando una propuesta es aprobada/rechazada, notificar a Hermes vía webhook para que lo sepa en Telegram.

**Files:**
- Create: `backend/src/proposals/proposal-notifier.service.ts`
- Add env var: `HERMES_WEBHOOK_URL`

Notifica a Hermes cuando:
- Proposal approved → `{ type: 'proposal.approved', proposalId, actionType, resultSummary }`
- Proposal rejected → `{ type: 'proposal.rejected', proposalId, actionType, rejectionReason }`
- Proposal executed → `{ type: 'proposal.executed', proposalId, actionType, result }`
- Proposal failed → `{ type: 'proposal.failed', proposalId, actionType, error }`
